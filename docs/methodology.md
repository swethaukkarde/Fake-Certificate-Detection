2 Methodology

The proposed system detects forged certificates using a deep learning–based approach. The system processes certificate images, extracts meaningful visual features, and classifies them as genuine or fake using a MobileNetV2 model.

The methodology consists of the following stages.

1. Data Collection

The first step involves collecting certificate samples for training and testing the model.

Genuine certificates are collected from available academic or sample documents.

Fake certificates are generated by modifying genuine certificates using image editing tools.

Common manipulations include changing text, replacing photos, altering seals, or modifying signatures.

Certificates may be provided as PDF documents or image files (JPG/PNG).

These certificates are organized into two categories:

dataset/
   real/
   fake/
2. Data Preprocessing

Before training the model, the certificate images must be preprocessed.

PDF certificates are converted into image format using pdf2image.

Images are resized to 224 × 224 pixels, which is the required input size for MobileNetV2.

Pixel values are normalized to improve training performance.

The dataset is split into training and validation sets.

Preprocessing ensures that the images have a consistent format suitable for deep learning models.

3. Feature Extraction

Feature extraction is performed using a Convolutional Neural Network (CNN).

The system uses MobileNetV2, a lightweight deep learning architecture designed for efficient image classification.

MobileNetV2 extracts important visual characteristics such as:

Certificate layout and structure

Font styles and alignment

Institutional logos

Stamps or seals

Background textures

Image patterns

These visual features help the model distinguish between authentic and manipulated certificates.

4. Model Training

After preprocessing, the dataset is used to train the deep learning model.

MobileNetV2 is used as a pre-trained base model.

The top classification layer is modified for binary classification (Real / Fake).

The base layers are frozen to retain learned visual features.

Additional layers such as Global Average Pooling, Dense, and Dropout are added to improve classification performance.

The model is trained using:

Binary Cross-Entropy Loss

Adam Optimizer

Training is performed for multiple epochs until the model learns to distinguish between real and fake certificates.

5. Model Evaluation

The trained model is evaluated using the validation dataset.

The following metrics are used to measure performance:

Accuracy – proportion of correctly classified certificates

Precision – correctness of positive predictions

Recall – ability to detect fake certificates

F1-score – balance between precision and recall

These metrics help determine how well the system detects forged certificates.

6. Certificate Prediction

After training, the system can analyze new certificate images.

The process includes:

Upload a certificate image.

Resize and normalize the image.

Pass the image through the trained MobileNetV2 model.

Generate a prediction score.

If the score is greater than the threshold, the certificate is classified as Genuine; otherwise, it is classified as Fake.

7. Output Generation

The final output of the system is the classification result:

Real Certificate

Fake Certificate

The system provides a fast and automated way to verify certificates and can be extended to support real-world document verification systems.
